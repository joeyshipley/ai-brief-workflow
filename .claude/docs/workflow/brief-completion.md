# Brief Completion

**Purpose:** Close a brief with explicit learning extraction and rule promotion check.

---

## Steps

### 1. Final Review

Read the entire brief. Confirm:
- [ ] All Success Signal items are checked
- [ ] All phases are marked Complete
- [ ] Open Questions are either answered or explicitly deferred

### 2. Learning Extraction

Fill in the brief's **Learning Log** section. Ask:
- "What surprised us during this brief?"
- "What took longer or shorter than we expected?"
- "What would we do differently next time?"
- "What behaviors should change going forward?"

### 3. Rule Promotion Check

Fill in the brief's **Rule Candidates** section. Ask:
- "Did this brief reveal a principle that should become a project rule?"
- "Did it reveal a workflow improvement that should change how briefs work?"
- "Should anything be added to the project's config checks?"

If the user confirms a rule candidate:
- Project rules -> add to the appropriate project doc (README, CLAUDE.md, or `.workflow/config.md`)
- Workflow rules -> note for the user to update the portable layer across projects

### 4. Mark Complete

- Set brief Status to **Complete**
- Add a final Session Log entry summarizing the completion
- Move the brief to the project's completed briefs directory (per `.workflow/config.md`)
