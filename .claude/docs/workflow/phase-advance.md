# Phase Advance

**Purpose:** Transition between phases with explicit checkpoint comparison and forward-look.

---

## Steps

### 1. Read the Brief

Identify the phase that just completed and the next phase.

### 2. Checkpoint Comparison

Read the completed phase's **Checkpoint** (what we expected) and **Done** (what happened). Surface any divergence:

> "The checkpoint assumed X. What actually happened was Y. Does this affect what comes next?"

### 3. Phase Table Review

Look at the remaining phases. Based on what the completed phase revealed:
- Do any later phase goals need rewording?
- Should any phases be added, removed, or reordered?
- Present proposed changes to the user for confirmation.

### 4. Begin Next Phase

Follow `.claude/docs/workflow/phase-workflow.md` to start the next phase (write its checkpoint, begin research).
